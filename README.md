# בוט פידבק למסעדות — WhatsApp

בוט רב-מסעדתי (multi-tenant): מספר WhatsApp אחד משרת את כל המסעדות. כל מסעדה
מזוהה לפי `org_id` במסד הנתונים (Supabase), והודעות נכנסות מנותבות אליה אוטומטית.

## איך זה עובד

1. **בעל המסעדה** שולח לבוט צילום מסך של הזמנה (טלפון + שם לקוח).
   הבוט מזהה את הפרטים (Gemini אם מוגדר מפתח, אחרת OCR מקומי) ומתזמן הודעת פידבק.
2. **אחרי ההשהיה** (מוגדרת פר-מסעדה) הבוט שולח ללקוח תבנית עם כפתורים.
3. **חוויה טובה** → הבוט שולח קישור דירוג בוולט של המסעדה.
   **"לפנות למנהל"** → הבוט אוסף את התלונה ומעביר למנהל המסעדה הרלוונטית.

## ניתוב רב-מסעדתי

- הודעה נכנסת ממספר שרשום ב-`org_phones` → זרימת **בעלים** של אותה מסעדה.
- הודעה מלקוח: לחיצת כפתור נושאת `context.id` (מזהה ההודעה המקורית) →
  איתור מדויק של השורה ב-`feedbacks` לפי `wa_message_id`. טקסט חופשי מנותב
  לפי המשוב הפעיל האחרון של אותו מספר.

## מבנה מסד הנתונים (Supabase, פרויקט `restaurant-feedback-bot`)

| טבלה | תפקיד |
|---|---|
| `orgs` | מסעדה: שם, מנהל, קישור וולט, שם תבנית, השהיה בדקות |
| `org_phones` | טלפונים מורשים לשליחת לקוחות (מפתח הניתוב) |
| `feedbacks` | כל משוב: לקוח, סטטוס, תוצאה, תלונה, wamid |
| `org_stats` (view) | סטטיסטיקות לדשבורד |

## דף ניהול

`http://localhost:3000/admin` — כניסה עם `ADMIN_KEY` מה-`.env`.
הוספת/עריכת מסעדות, צפייה במשובים, סטטיסטיקות, שליחה ידנית וביטול.

## הוספת מסעדה חדשה

1. בדף הניהול: "+ הוספת מסעדה" — שם, טלפון מנהל, קישור וולט, טלפונים מורשים.
2. ב-Meta Business Manager: לוודא שיש תבנית הודעה מאושרת עבורה (אפשר להשתמש
   בתבנית משותפת או ליצור ייעודית) — שם התבנית מוגדר פר-מסעדה בדשבורד.
   מבנה התבנית: פרמטר header = שם המנהל, פרמטר body = שם הלקוח, כפתורי
   quick-reply (חיובי / פנייה למנהל).

## הרצה

```bash
npm install
npm run build
npm start          # שרת + מתזמן, פורט 3000
npm run test:e2e   # בדיקות קצה-לקצה מול Graph API מדומה (לא שולח הודעות אמיתיות)
```

ה-webhook של Meta צריך להצביע על `https://<domain>/webhook` (בפיתוח: ngrok/cloudflared).

## משתני סביבה (.env)

| משתנה | תיאור |
|---|---|
| `WHATSAPP_TOKEN` | טוקן קבוע (System User) — לא טוקן זמני של 24 שעות! |
| `WHATSAPP_PHONE_NUMBER_ID` | מזהה המספר המשותף |
| `WEBHOOK_VERIFY_TOKEN` | מחרוזת אימות ה-webhook מול Meta |
| `SUPABASE_URL`, `SUPABASE_KEY` | חיבור למסד הנתונים |
| `ADMIN_KEY` | סיסמת דף הניהול |
| `GEMINI_API_KEY` | אופציונלי — קריאת שם+טלפון מתמונות (מפתח `AIza...`) |
| `POLL_INTERVAL_MS` | תדירות בדיקת משובים לשליחה (ברירת מחדל 20 שניות) |
